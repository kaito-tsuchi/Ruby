# Ruby

# 目的
「ゼロから分かるRuby超入門　著:五十嵐邦明、松岡浩平」の参考書を元に、Rubyの勉強を行う。

#スケジュール
2/1：環境構築

windowsにRubyをインストールし、VSコードでコードを書き、コマンドプロンプトで実行するように環境を整えた。また、VSコードをGitHubに連携した。

2/2～2/3：Rubyの基本的な知識

まず、Rubyの基礎知識として、数値や文字列の表示方法や四則演算、オブジェクト、変数について簡単なコードを書きつつ学んだ。この単元は、今まで学んだCやJavaなどの知識を活かすことができたので、比較的簡単だった。

2/4：条件分岐

ifやunless、else節、case、timesメソッド、whileの書き方や機能をコードを書きつつ学んだ。この単元も、今まで学んだ言語の知識を活かすことができたので、スムーズに学習することができた。しかし、timesメソッドについては、初めて学んだ内容だった。このメソッドは、決まった回数だけ処理を繰り返すことができるので、とても便利で使い道も多そうだと思った。

2/5：配列

配列について、要素の取得や追加、削除から配列同士の演算、繰り返し処理の方法を学んだ。特に印象的なのは、eachメソッドを用いた繰り返し処理で、途中で終わらせるbreakや繰り返しの次の回に進むnext、範囲を指定する（）等、初めて学ぶことが多くあった。ここの部分はよく復習しておきたい。

2/6：メソッドの利用

ここの単元では、Rubyに用意されている様々なメソッドの調べ方を学んだ。同じ名前のメソッドでも末尾に！がつくと機能が変化することがあるのがとても印象的だった。この単元で用いたのは、配列の要素数を得る -sizeメソッド、配列の全要素の合計を得る -sumメソッド、配列から重複した要素を取り除く -uniqメソッド、ランダムに要素を取得する　－sampleメソッド、配列の要素を順に並び変える -sortメソッド、配列の要素を逆順にする -reverseメソッド、配列中の文字列を連結する -joinメソッド、文字列を分割して配列にする -splitメソッド、配列の各要素を変換した配列を作る -mapメソッド等である。

2/7：ハッシュ

この単元では、ハッシュという複数のオブジェクトをまとめて入れることができる道具について学んだ。ハッシュは「キー」と「値」のセットで、複数のデータを整理し扱っており、データを効率よく操作することができるのが特徴である。ハッシュの定義方法や名前の付け方、ハッシュから値を取得する方法、キーと組みの追加、削除方法、ハッシュの要素の繰り返し処理の方法などを主に学んだ。

2/9：メソッドを用いたプログラムの分割

この単元では、メソッドの意味や定義方法、呼び出し方法や戻り値について学んだ。メソッドとは、処理のかたまりに名前を付けたもので、defに続いて処理を書くことで定義できる。定義したメソッドは呼び出すことで実行で実行でき、メソッドで最後に実行した結果が戻り値となる。キーワード引数を使うと、メソッドの呼び出し時に引数を名前付きで指定できる、また、変数にはスコープ（見える範囲と寿命）があり、ローカル変数は定義したメソッドの中がスコープであり、メソッドの外からは見えない。同様に、メソッドの外で定義したローカル変数はメソッドの中からは見えない。

2/10～2/11：クラス

この単元では、主にオブジェクトやクラス、インスタンスの概念について学んだ。まずオブジェクトとは、データとそれに対する操作を持つ存在であり、Rubyは全てがオブジェクトで表されている。クラスはオブジェクトの種族を表すものであり、全てのオブジェクトはいずれかのクラスに属している。クラスの作成方法やクラス中のメソッドの定義方法、クラス中のメソッドの呼び出し方法なども学んだ。レシーバは、メソッドが呼び出されるオブジェクトのことであり、例えば、2.even?のレシーバは2である。

また、名前が@で始まる変数をインスタンス変数と呼び、ローカル変数よりもスコープが広く同じオブジェクトであればメソッドをまたいで使うことができる。このインスタンス変数はオブジェクトごとに持っており、代入が実行されたときに生まれ、オブジェクトと同じ寿命を持つ。

クラスに作るメソッドは2種類あり、レシーバがインスタンスでありインスタンスに対して呼び出すインスタンスメソッドと、レシーバがクラスでありクラスに対して呼び出すクラスメソッドがある。このクラスメソッドの定義方法は、def self メソッド名の後に処理を書く。

子クラスは親クラスの全てのメソッドを「継承」でき、superを用いると親クラスの同名のメソッドを呼び出すことができる。

クラス中でprivaeよりも後ろで定義したメソッドはprivateメソッドとなり、レシーバを指定したオブジェクト.メソッド名の形式でしか呼び出しができなくなる。また、クラスの中でprivateよりも前またはprivateを書かずに定義したメソッド及び、publicよりも後ろで定義したメソッドはpublicなメソッドとなり、オブジェクト.メソッドの形式でもレシーバを指定しないメソッド名の形式でも呼び出しができる。

2/12：モジュール

この単元では、モジュールを定義し、クラスにモジュールをインクルードすることで、メソッドを共同利用することができると学んだ。モジュールはインスタンスメソッドを定義でき、クラスと異なりインスタンスを作ることはできない。また、モジュールはクラスメソッド、変数を定義でき、クラス中で変数を使うときは::でモジュールと定数名をつなぐ。
require_relativeメソッドを使うと、別ファイルに定義されたクラスやモジュールを読み込んで使用できる。
